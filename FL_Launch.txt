1. Levantar todo
bashdocker compose up -d
2. Verificar que todos los contenedores están arriba
bashdocker compose ps
3. Ver que el Consumer arrancó y el poller está esperando
bashdocker logs be-dataapp-consumer | grep -E "fl-poller|algoritmo|algorithm|poller"
4. Verificar que los workers tienen el algoritmo cargado
bashcurl http://localhost:5001/status
curl http://localhost:5002/status
curl http://localhost:5003/status
Los tres deben tener "algorithm_loaded": true.
5. Lanzar el FL
bashcurl -X POST http://localhost:8600/fl/start \
     -H "Content-Type: application/json" \
     -d '{"rounds": 5}'
6. Monitorear rondas en tiempo real
bashdocker logs -f be-dataapp-provider | grep -E "RONDA|Ronda [0-9]+ OK|completado"
7. Cuando termine, ver el poller actuar
bashdocker logs be-dataapp-consumer | grep -E "fl-poller|IDS-MODEL|IDS-RESULTS|CICLO|fallback"
8. Verificar resultado final en el Consumer
bashcurl http://localhost:8501/status
curl http://localhost:8501/fl/model
curl http://localhost:8501/fl/results/summary
Si en el paso 8 ves model_received: true y results_received: true, el flujo completo funciona.

Fase 4 — Probar UNA sola ronda completa (coordinador + 3 workers)
bashcurl -X POST http://localhost:8600/fl/round
Esto ejecuta exactamente una ronda: coordinador llama a los 3 workers en paralelo, hace FedAvg, guarda el modelo global. Respuesta esperada:
json{
  "round": 1,
  "workers_ok": 3,
  "total_samples": 54000,
  "global_metrics": {
    "loss": 0.43,
    "accuracy": 0.84,
    "auc": 0.90,
    "precision": 0.81,
    "recall": 0.87
  }
}
Lo crítico aquí es "workers_ok": 3 — si es menos de 3, algún worker falló.

Fase 5 — Lanzar el FL completo (5 rondas)
bash# Lanzar
curl -X POST http://localhost:8600/fl/start \
     -H "Content-Type: application/json" \
     -d "{\"rounds\": 5}"

# Monitorizar cada 10 segundos
curl http://localhost:8600/fl/status

# Ver resultados cuando termine
curl http://localhost:8600/fl/results
```

**Lo que debes observar** en los resultados para confirmar que FL funciona correctamente: la `accuracy` y el `auc` deben **subir ronda a ronda**. No tiene que ser dramático, pero sí una tendencia ascendente. Por ejemplo:
```
Ronda 1 → accuracy: 0.82, auc: 0.88
Ronda 2 → accuracy: 0.85, auc: 0.91
Ronda 3 → accuracy: 0.87, auc: 0.93
...
Si las métricas suben, el sistema FL está funcionando correctamente: los workers están aprendiendo de forma colaborativa sin compartir datos.
